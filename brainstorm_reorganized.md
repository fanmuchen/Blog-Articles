## Agent Types

Agents should be classified into 2 types:

### 1) Operational Agents

- Read memory and skills.
- Call tools.
- Basically treated as a complete person.

### 2) Steward Agents

- Have premade goals and call dedicated tools.
- Run tidying, evaluating, and summing-up jobs.
- Run as a "sidekick".
- Usually one-round calls unless orchestrated.
- Example: read every AI-human interaction to determine what relational memory to write into a character's memory.

## Cycle and State Machine

Characters can be `sleeping` / `busy` / `idle`:

- `sleeping` -> do not respond to any chat
- `busy` -> use steward agents to generate a one-time response
- `idle` -> normal

## Tool Call Auditing

All tool calls should be audited:

- time of execution
- success or failure
- reason for getting stuck

These records should be collected for tool improvement.

## Context Layering

Chat logs between human users and the system should be a side product of Operational Agents.

The context of Operational Agents contains:

- system instructions
- tool calls
- agents' thinking

What is finally output and observable by users should be generated by tool calls.

When concatenating for future use (for example, summarizing into historical memory and generating continuous chat messages in later interactions), include only:

- what was actually said in chat
- a brief summary of tool call results
